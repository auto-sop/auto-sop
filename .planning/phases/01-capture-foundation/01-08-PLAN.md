Implementation notes:
- Create an isolated tmp project root via `fs.mkdtempSync`. This becomes `<PROJECT_ROOT>` in the fixture.
- Stub a minimal transcript file at `<tmp>/transcript.jsonl` with a single assistant message block; use for `<FIXTURE_TRANSCRIPT>`.
- Set `HOME=<tmp>/fake-home` via env so the global mirror lives under the isolated fs (keeps tests parallel-safe). The writer uses `os.homedir()` → honors HOME.
- For each line: substitute placeholders, spawn `node dist/capture/shim.cjs` with stdin = the JSON line, wait for exit. The writer is detached; don't wait for it here.
- After each line: no sleep — pump them as fast as possible. Quiescence waits at the end.
- `waitForQuiescence`: poll every 50ms for (a) no `*.pending` dirs + (b) tmp dir empty or only fresh files. Timeout 5s; if exceeded, fail with diagnostics (list remaining .pending contents).
- `preActions`: run before any fixture lines are piped. `create-paused-flag` writes the flag file. `age-pending-dirs` — not usable in preActions since nothing exists yet.
- `midActions`: applied when a `{"_scenario_marker":"BREAK_HERE"}` line is encountered. `age-pending-dirs` recursively calls `utimesSync` on every file inside each `*.pending` dir in `captureDir`, setting mtime to `now - minusSeconds`.
- When env is set, merge it into process.env for the spawned shim.

Avoid: Do NOT use memfs here — we want to test the actually-built binary against the real filesystem, matching production. Do NOT share HOME/project dirs across scenarios (race + test pollution). Do NOT add sleeps; use polling in waitForQuiescence.
