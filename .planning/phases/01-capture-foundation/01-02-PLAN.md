Create `src/capture/shim/main.ts` following RESEARCH.md §Pattern 1 ("Hot-path shim") EXACTLY. The shim must:

1. Check `isCaptureDisabled(process.env)` from `../kill-switch.js` FIRST. If disabled: drain stdin to avoid SIGPIPE on the CC parent, then `process.exit(0)`. Never write any file, never spawn anything.
2. If enabled: read stdin into a `Buffer[]` accumulator, concatenate on `'end'`.
3. Compute `tmpRoot = join(homedir(), '.claude-sop', 'tmp')`, `mkdirSync(tmpRoot, {recursive:true, mode: 0o700})`.
4. Generate `tmpName = nanoid(16) + '.json'`, `writeFileSync(tmpPath, payload, {mode: 0o600})`.
5. Call `handoffToWriter(tmpPath)` from `./handoff.js`.
6. `process.exit(0)`.

Imports allowed in main.ts: ONLY `node:child_process`, `node:fs`, `node:path`, `node:os`, `nanoid`, and `../kill-switch.js` + `./handoff.js`. DO NOT import from src/config, src/scrubber, src/path-resolver, zod, yaml, execa, proper-lockfile, or anything else. Use `node:` prefixes everywhere.

Create `src/capture/shim/handoff.ts` exporting a `spawnWriter` that takes the writer entry as a required argument — NO env var branches, NO test-only overrides in the production shim:
```ts
import { spawn } from 'node:child_process';

export function spawnWriter(payloadPath: string, writerEntry: string): void {
  const child = spawn(process.execPath, [writerEntry, payloadPath], {
    detached: true,
    stdio: 'ignore',
  });
  child.unref();
}
```

Create `src/capture/shim/shim-config.ts` containing the single baked-in constant the production shim uses:
```ts
import { join } from 'node:path';
export const WRITER_ENTRY = join(__dirname, 'writer.cjs');
```
(tsup bundles this into shim.cjs; the string is baked in at build time.)

`main.ts` imports `WRITER_ENTRY` from `./shim-config.js` and calls `spawnWriter(tmpPath, WRITER_ENTRY)`. **The production shim contains zero test branches and reads zero env vars beyond the kill-switch.** (B3 fix: removes `CLAUDE_SOP_WRITER_OVERRIDE`.)

Tests import `spawnWriter` directly and pass a fake writer path — no env var plumbing required.
